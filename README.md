# Margin

Margin is a document-like review layer for Markdown files in GitHub repositories. Business users comment on rendered Markdown and publish finished reviews into Git-native artifacts.

## Stack

- Svelte 5 + TypeScript + Vite SPA
- Tauri v2 desktop shell
- Rust API with Axum + Tokio
- Postgres via SQLx migrations
- OpenAPI generated by the Rust service

## Run Locally

Install dependencies:

```sh
just setup
```

Start the API:

```sh
just dev-api
```

Start the web app:

```sh
just dev-web
```

Open `http://127.0.0.1:5173`.

The API runs without `DATABASE_URL` by using in-memory review state. Set `DATABASE_URL` to connect Postgres for the planned durable storage path.

## Build and Deploy

Margin uses [`just`](https://github.com/casey/just) as the project CLI. Run `just` to list commands.

```sh
just check
just test
just build
just deploy local
```

`just build` builds the Rust API and Svelte web app. `just build-all` also builds the Tauri desktop app. Remote deploy targets such as `staging` and `production` are reserved in the `justfile` but intentionally fail until hosting infrastructure is configured.

## Desktop

```sh
just dev-desktop
```

The desktop app is a Tauri wrapper around the same Svelte SPA.

## API Contracts

With the API running:

```sh
just api-types
```

This fetches `http://127.0.0.1:4317/openapi.json` and writes TypeScript definitions into `apps/web/src/lib/generated/api.d.ts`.

## Product Shape

The current vertical slice includes:

- rendered Markdown review surface
- selected-text comments
- selected-text replacement suggestions
- robust anchor payloads with lines, quote, context, heading path, and content hash
- publish preview with patched Markdown and sidecar YAML
- SSE heartbeat endpoint for live-ish review updates

GitHub App PR creation is intentionally left as the next integration boundary; the publish endpoint already returns the branch, sidecar, and patch data that boundary will consume.
